Limitations

Fantasy Consoles like Pico-8 are
designed to have drawbacks.

To the left and right of this paragraph are Pico-8 cartridges stored in PNG images. This extraordinary
feat of data compression is one of the core features of Pico-8, accomplished by the fact that there is a
limited amount of sprites, sounds, and code for the developer to work with.

Various limitations placed on the
developer environment are listed below.

Data Type Data limit
Characters 65536
Tokens 8192
PNG size ~16 KB

These limitations are put in place to accomplish a few things.

  1. Encourage developers to write their code as efficiently as possible,
    even if it always doesn't result in the most readable code.
    In the case of "Tokens" (variable or function names), they are put in place to
    force developers to write functions that can accomplish the most amount
    of tasks with the least amount of variables or functions as possible.
  2. Allow for games to be exported into PNG images which can easily
    be shared throughout the internet due to their small size.
  3. Allow for games to be implemented into HTML5 websites with little
    to no performance issues on the user's end.
